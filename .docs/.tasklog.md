# Project Modification Work Log

This document tracks all significant modifications, fixes, and enhancements applied to the "Instant Book Exchange" project to prepare it for final presentation.

## 1. Infrastructure & Deployment

### Docker Configuration
*   **Fix:** Removed obsolete `version` attribute from `docker-compose.yml`.
*   **Fix:** Resolved `permission denied` error in `docker-entrypoint.sh` by adding `chmod +x` in the Dockerfile.
*   **Enhancement:** Renamed `docker-compose-local.yml` to the standard `docker-compose.yml` for ease of use.
*   **New Feature:** Created `scripts/setup_env.sh` to automate the installation of Docker, Make, and Git on Ubuntu/Debian systems.

### Deployment Automation
*   **New Feature:** Created a robust `scripts/deploy.sh` script (executed via `make deploy`).
    *   **Auto-Cleanup:** Automatically runs `docker-compose down` to remove zombie containers.
    *   **Pre-flight Checks:** Verifies Docker daemon is running and ports (3000, 8000, 5432) are free.
    *   **Health Polling:** Explicitly waits for the Database to allow connections before reporting success.
*   **Enhancement:** Added `make dev` and `make up` targets to the Makefile for improved developer workflow, orchestrating all Docker services (DB, Web, Frontend) in one command.

## 2. 🔧 Backend Fixes (Django)

### Critical Error Resolution
*   **Fix (500 Internal Server Error):** Configured the `console` email backend. The app was crashing when trying to send emails (e.g., verification) because no SMTP server was configured.
*   **Fix (Permissions):** Updated `TransactionsView` to require `IsAuthenticated`. Anonymous users were causing crashes by triggering logic dependent on `request.user`.
*   **Fix (Verification):** Refined database health checking logic in the deployment scripts.

## 3. Frontend Fixes (Nuxt.js)

### State & Networking
*   **Critical Fix (Hydration Error):** Refactored Pinia stores (`dataStore`, `userStore`, `profileStore`).
    *   **Issue:** The server (SSR) was hydrating the state with the *internal* Docker URL (`http://web:8000`), which then broke client-side requests (browsers can't resolve `web`).
    *   **Solution:** Moved `BE_API` logic to getters, ensuring the client always uses `runtimeConfig.public.apiBase` (`http://localhost:8000`).

### UI/UX
*   **Mobile Responsiveness:** Refactored the Navigation Bar to use a "Hamburger Menu" on mobile devices (previously, links were inaccessible).
*   **Navigation:** Moved "About" and "Support" links from the footer to the main Navbar for better visibility.
*   **Cleanup:** Removed the footer as requested.
*   **New Feature:** Added **Edit** and **Remove** buttons directly to the **Book Details** page for book owners.
    *   Integrated with `Addbook.vue` for pre-filled edit forms.
    *   Implemented secure DELETE logic with user confirmation.

## 4. Code Structure Refactoring

### Monorepo Reorganization
*   **Standardization:** Converted the flat file structure into a clean Monorepo layout:
    *   `backend/`: Contains all Django apps (`authentication`, `transactions`, etc.).
    *   `frontend/`: Renamed from `Boookz-Nuxt`.
    *   **Root:** Contains only orchestration files (`docker-compose.yml`, `Makefile`) and documentation.
*   **Cleanup:** Deleted unused files: `package.json` (root), `ToDos.md`, `notes.md`, and redundant `staticfiles`.
